# RunTube — 現在仕様

> エクササイズバイクの走行速度に応じて YouTube を自動再生・停止する、単一 HTML の Web アプリ

---

## 概要

RunTube は、Web Bluetooth でエクササイズバイクから速度データを取得し、閾値を超えている間だけ YouTube 動画を再生する静的サイトである。  
アプリ本体は `index.html` の単一ファイルで完結し、GitHub Pages で配信する。

現行実装では、単純な閾値再生だけでなく、以下も含む。

- 前回接続したデバイスの再接続
- Auto インターバルモード
- 走行日次ログとカレンダー表示
- 再生動画の切り替えと履歴保存

## 機能要件

### 1. Bluetooth 接続

| 項目 | 内容 |
|---|---|
| 対象デバイス | `QB` / `NEXGIM` / `C01` の name prefix を持つバイクを主対象とする |
| 接続方法 | `バイク接続`、`すべてのデバイス`、`前回のデバイスに接続` の 3 系統 |
| 使用サービス | FTMS (`0x1826`)、Cycling Speed and Cadence (`0x1816`)、Cycling Power (`0x1818`) |
| 切断検知 | `gattserverdisconnected` で切断を検知し、UI と再生状態を初期化する |
| 前回接続保存 | 接続成功時にデバイス名と先頭 3 文字の prefix を `localStorage` に保存する |

接続ボタンの仕様は以下の通り。

- `バイク接続`: `QB` / `NEXGIM` / `C01` の prefix でフィルタした `requestDevice()` を呼ぶ
- `すべてのデバイス`: `acceptAllDevices: true` で接続ダイアログを開く
- `前回のデバイスに接続`: 保存済み prefix で再フィルタして接続ダイアログを開く
- `切断`: `device.gatt.disconnect()` 実行後にアプリ側状態を明示的に初期化する

### 2. 取得データ

#### 2.1 FTMS / Indoor Bike Data

`Fitness Machine` サービスの `Indoor Bike Data` (`0x2AD2`) を第一優先で購読する。

- 速度: `Instantaneous Speed` を `0.01 km/h` 単位で解釈
- ケイデンス: フラグに応じて読み取り、`0.5 rpm` 単位を補正
- パワー: フラグに応じて `int16` で読み取り

### 2.2 Cycling Power

`Cycling Power Measurement` (`0x2A63`) を購読し、取得できる場合はパワー表示を更新する。

### 2.3 CSC フォールバック

`CSC Measurement` (`0x2A5B`) は速度の代替計算には使わず、ケイデンス表示の補完として使用する。

- クランク回転数差分とイベント時刻差分から rpm を算出
- FTMS 速度が主制御入力であり、再生/停止判定は `currentSpeed` に対してのみ行う

### 3. YouTube 再生

YouTube IFrame Player API を使って動画を埋め込み、JavaScript から再生制御する。

デフォルト動作:

- 初期動画は `FIRST_VIDEO`
- 初期動画のまま起動した場合はプレイリスト `PLAYLIST_ID` を紐づけてロード
- プレーヤー操作は無効
  - `controls: 0`
  - `disablekb: 1`
  - プレーヤー要素に `pointer-events: none`
  - `fs: 0`

オートプレイ制限対策として、プレーヤー準備完了時に以下を一度だけ行う。

1. ミュート
2. 再生
3. 一時停止
4. ミュート解除

### 4. 動画切り替え

デフォルトプレイリスト以外にも、任意の YouTube 動画へ切り替えられる。

| 方法 | 内容 |
|---|---|
| URL パラメータ | `?v=VIDEO_ID` または YouTube URL から初期動画を切り替える |
| モーダル入力 | `動画を変更` ボタンから URL または動画 ID を入力して再生する |
| 履歴再生 | 保存済み動画履歴から再読み込みする |

対応する入力形式:

- 11 文字の動画 ID
- `youtu.be/...`
- `youtube.com/watch?v=...`
- `youtube.com/shorts/...`
- `youtube.com/embed/...`

動画切り替え時の仕様:

- `player.loadVideoById(videoId)` で切り替える
- 読み込み直後に一時停止し、実際の再生は速度判定に従う
- タイトルは YouTube oEmbed から取得し、失敗時は動画 ID をそのまま使う

### 5. 再生制御

再生制御は Bluetooth 接続中のみ有効とする。

```text
未走行かつ 速度 >= 閾値 + 1.0   -> 再生開始
走行中かつ 速度 <  閾値 - 1.0   -> 一時停止
```

補足:

- ヒステリシスは `±1.0 km/h`
- 閾値プリセットは `30 / 35 / 40 / 50 km/h`
- デフォルト閾値は `30 km/h`
- Auto モード中は手動の閾値変更を無効化する

### 6. Auto インターバルモード

`Auto` ボタンでインターバルトレーニングモードを開始/停止できる。

デフォルトフェーズ:

| フェーズ | 時間 | 目標速度 |
|---|---|---|
| ウォームアップ | 3 分 | 30 km/h |
| 高負荷 | 2 分 | 40 km/h |
| 休憩 | 1 分 | 30 km/h |
| 高負荷 | 2 分 | 45 km/h |
| 休憩 | 1 分 | 30 km/h |
| 高負荷 | 2 分 | 50 km/h |
| クールダウン | 2 分 | 30 km/h |

モード中の挙動:

- 現在フェーズの `targetKmh` を `thresholdKmh` に反映
- 経過秒数に応じて残り時間、セット数、進捗バーを更新
- フェーズ切り替え時に全画面フラッシュ演出で目標速度を大きく表示
- 全フェーズ完了時は完了オーバーレイを 4 秒表示
- 終了時は通常モードへ戻し、閾値を `30 km/h` に戻す

### 7. Auto 最適化

Auto モード終了時、十分な走行サンプルがあれば平均速度を保存し、次回以降の Auto フェーズを自動調整する。

保存条件:

- `isRiding === true` の間だけ速度サンプルを収集
- サンプル数が 10 件以上ある場合のみセッション平均を保存

最適化ロジック:

- 直近 5 セッション平均の平均値を算出
- ベース閾値 = `round(avg * 0.80)` を `25..48` に clamp
- デフォルト基準値 `30` との差分を全フェーズへ加算
- 各フェーズ閾値は `20..55` に clamp

UI:

- 保存済みプロファイルがある場合、`⚡ 最適化済み XX km/h〜（N回分）` を表示
- リセットボタンで速度プロファイルを削除できる

### 8. 速度低下時の演出

速度が閾値未満になった場合、動画停止に加えてアイリス演出を行う。

- 停止時: `iris-out` でプレーヤー領域を円形に縮小
- 再開時: `iris-in` で表示を復帰
- 停止中オーバーレイには `ペダルを漕いでください` と必要速度を表示

### 9. 走行ログとカレンダー

走行中は 1 秒ごとに当日の走行ログを `localStorage` に蓄積する。

保存内容:

- 走行秒数 `sec`
- 平均速度 `avgKmh`
- 平均計算用サンプル数 `samples`

旧形式の数値のみ保存されたエントリも読み込み時に互換変換する。

カレンダー仕様:

- 月表示の前後移動が可能
- 走行日のセルを 4 段階で着色
- セルに走行時間と平均速度を表示
- 今日のセルは 1 秒ごとの更新で部分反映
- 連続走行日数を `🔥 N日連続` で表示

走行レベルの閾値:

- `0 < sec < 5分` -> `lvl1`
- `5分 <= sec < 15分` -> `lvl2`
- `15分 <= sec < 30分` -> `lvl3`
- `30分以上` -> `lvl4`

### 10. 動画履歴

再生した動画は履歴として最大 20 件保存する。

- 同一動画は先頭に再配置し、重複保存しない
- サムネイルは `https://i.ytimg.com/vi/<id>/default.jpg`
- 現在再生中の動画は履歴上で強調表示する
- `クリア` ボタンで履歴を全削除できる

### 11. UI 状態

主な表示状態は以下の通り。

| 状態 | 表示 |
|---|---|
| 未接続 | オーバーレイ `Bluetooth で接続してください` |
| 接続済み・停止中 | オーバーレイ `ペダルを漕いでください` |
| 接続済み・走行中 | オーバーレイ非表示、動画再生 |
| 切断 | 表示値を初期化し、未接続表示へ戻す |
| Auto 実行中 | インターバルバナー表示、閾値ボタン無効化 |

レスポンシブ仕様:

- モバイルではプレーヤー下に操作群を表示
- `768px` 以上では右サイドバー構成へ切り替える
- モバイル用とサイドバー用で同等の操作 UI を重複配置し、同一 state で同期更新する

## 非機能要件

- 単一 HTML ファイル構成で、ビルド手順は持たない
- GitHub Pages で配信可能な静的サイトであること
- Web Bluetooth 利用のため `HTTPS` または `localhost` が必要
- 対応ブラウザは Chrome / Edge 系
- iOS Safari は非対応前提

---

## 診断ページ

`qb_c01s_web_bluetooth_test.html` は、本番 UI とは独立した Bluetooth 診断ページである。

用途:

- デバイス接続可否の確認
- 見えている GATT サービス / characteristic の列挙
- 生データ確認
- FTMS / CSC / Cycling Power の取得可否確認

本番の `index.html` に実装している Bluetooth ロジックの確認用ページとして維持する。

## デプロイ

- `main` ブランチへの push で GitHub Pages に反映される
- CI やビルドパイプラインは持たない

---

## 現在の制約

- 再生制御は FTMS の速度値に依存しており、CSC から疑似速度は算出していない
- YouTube プレーヤーの手動操作はできない
- Web Bluetooth 非対応ブラウザでは利用できない
