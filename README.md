# 🚴 RunTube

エクササイズバイク（QB-C01S）で **走っている間だけ YouTube を見られる**ページ。  
速度が設定した閾値を超えると動画が再生され、下回ると一時停止します。

**→ [https://ryotana424.github.io/run-tube/](https://ryotana424.github.io/run-tube/)**

---

## 使い方

1. **Chrome または Edge** でページを開く（iOS Safari は非対応）
2. バイクの電源を入れる
3. 「バイク接続」ボタンをタップしてペアリング
4. 再生閾値を **30 / 35 / 40 / 50 km/h** から選択、または **Auto モード**を使う
5. ペダルを漕いで閾値を超えると動画が自動再生 🎬

2回目以降は「前回のデバイスに接続」ボタンが表示され、ワンタップで接続できます。

---

## Auto モード

「Auto」ボタンを押すとインターバルトレーニングが始まります。

| フェーズ | 時間 | 目標速度 |
|---|---|---|
| ウォームアップ | 3分 | 30 km/h |
| 高負荷 1セット | 2分 | 40 km/h |
| 休憩 | 1分 | 30 km/h |
| 高負荷 2セット | 2分 | 45 km/h |
| 休憩 | 1分 | 30 km/h |
| 高負荷 3セット | 2分 | 50 km/h |
| クールダウン | 2分 | 30 km/h |

フェーズが切り替わるたびに、目標速度が画面に大きく表示されます。

### 閾値の自動最適化

Autoモード終了時に走行中の平均速度をローカルストレージへ保存（直近5セッション）。  
次回起動時、過去の平均から「少し頑張れば届く」ラインを自動計算してフェーズ全体をスケールします。

---

## 速度が下がったら…

閾値を下回ると動画が一時停止し、**アイリスアウト演出**で画面が縮小します。  
再び漕ぎ始めて閾値を超えると自動で再生が再開されます。

---

## 走行カレンダー

ページ下部にスクロールすると月次カレンダーが表示されます。

- 走行した日はグリーンで色付き（時間に応じて4段階の濃さ）
- 各セルに **走行時間** と **平均速度** を表示
- **連続走行日数**（🔥 N日連続）をリアルタイム表示
- データは `localStorage` に保存されます

## 仕組み

```
エクササイズバイク
      │  Web Bluetooth (FTMS / Indoor Bike Data)
      ▼
  速度 (km/h) を取得
      │
      ├─ 速度 ≥ 閾値 ＋ 1.0  →  YouTube 再生
      └─ 速度 ＜ 閾値 − 1.0  →  YouTube 停止 ＋ アイリスアウト
```

- **Bluetooth**: FTMS サービス (0x1826) の Indoor Bike Data を優先取得、CSC Measurement (0x2A5B) をフォールバックに使用
- **YouTube**: IFrame Player API で制御。再生ボタンは非表示・操作不可
- **前回デバイス記憶**: 接続成功時にデバイス名を `localStorage` へ保存
- **走行ログ**: `runtube_ride_log` に日付キーで秒数・平均速度を保存
- **速度プロファイル**: `runtube_speed_profile` に直近5セッションの平均速度を保存（Auto最適化用）
